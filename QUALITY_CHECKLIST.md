# Quality and Maintenance Checklist

Use this checklist when adding or reviewing content.

## Question File Quality
- [ ] One concept per file
- [ ] Question statement is clear and interview-relevant
- [ ] Answer explains what, why, and when
- [ ] Includes at least one realistic code snippet
- [ ] Keywords/tags are present and consistent
- [ ] Difficulty is correctly labeled
- [ ] Related questions are linked where applicable

## Topic Index Quality
- [ ] New question is linked in topic `README.md`
- [ ] Links are valid and relative
- [ ] Ordering follows learning flow (Foundational -> Core -> Advanced-in-topic)

## Repository Maintenance
- [ ] Outdated answers are updated with current best practices
- [ ] Duplicate or overlapping questions are consolidated
- [ ] Broken links are fixed
- [ ] New topics follow existing folder conventions

## Review Cadence
- Weekly: quick scan for broken links and missing index updates
- Monthly: validate technical accuracy of high-traffic topics
- Quarterly: add new questions for evolving frameworks and patterns
